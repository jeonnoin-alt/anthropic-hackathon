# SpendSentry 데모 가이드 (DEMO.md)

Sentri AI 지출결의서 컴플라이언스 챗봇. 이 문서는 **(1) 챗봇 최적화 내역**과
**(2) 모든 기능을 시연하는 데모 시나리오**(붙여넣기용 예시 + 예상 판정)로 구성됩니다.

---

## Part 1. 챗봇 최적화에 무엇이 들어갔나

| 영역 | 최적화 내용 | 효과 |
|------|------------|------|
| **멀티턴** | 정책을 system 프롬프트에 상주 + prompt caching, 대화 히스토리 유지 | 꼬리질문이 직전 결의서·영수증을 기억 |
| **응답 모드 분리** | 입력 의도 분류(전체검토 / 후속질문 / 정책질문 / 범위밖) | 간단한 질문에 전체 보고서 양식을 반복 출력하지 않음 |
| **범위 가드레일** | 지출·경비·세무 외 질문은 정중히 거절 | 뚱딴지 답변 방지 |
| **환각 방지** | 정보 부족 시 추측 대신 되묻기, 정책에 없는 규칙은 지어내지 않음 | 신뢰성 |
| **결정적 규칙 라우팅 (LangGraph)** | 규격화 질문은 LLM이 아니라 고정 Python 규칙으로 라우팅 | 같은 질문 → 항상 같은 답 |
| **결정적 규칙엔진** | R-01~R-12 한도·날짜·임계 판정을 순수 함수로 | 금액·날짜 계산 100% 일관 |
| **영수증 이미지 검증** | Claude 비전 추출 + 결정적 교차검증(금액·날짜·업종·증빙·야근택시 R-08) | "첨부 여부"가 아니라 "내용 일치" 검증 |
| **결제수단 결정적 분류** | 비전은 원문만 추출, '법인' 포함 여부는 코드가 판정 | '현대법인카드'를 개인카드로 오분류하던 버그 제거 |
| **추출 무결성** | Pydantic 스키마 검증 + 재시도 (`ReceiptExtract`) | JSON 파싱 에러/형식 오류 차단 |
| **속도** | 비전 전송 전 이미지 다운스케일(긴 변 ≤1568px), 모델 `claude-haiku-4-5`, max_tokens 축소 | 영수증 판독 ~3초 |
| **멀티모달 입력** | 한 채팅창에서 텍스트 + 영수증 이미지 + 엑셀(.xlsx) | 실무 입력 그대로 |
| **시각화** | PASS/FAIL 배너 · 지표 카드 · 심각도별 컬러 카드 | 데모 가독성 |

> 핵심 원칙: **LLM은 추출·대화만 담당하고, 판정·금액·날짜·임계 계산은 전부 결정적 코드**가 한다.

## Part 2. 데모 시나리오

> **입력 위치**: Streamlit 챗봇(`streamlit run app.py`)의 채팅창. 지출결의서는 통째로 붙여넣고,
> 질문은 그냥 입력. 영수증 이미지는 채팅창의 `+`(첨부)로 올립니다.

---

### A. 전체 지출결의서 검토 — 붙여넣으면 PASS / FAIL 판정

#### 🔴 A-1. FAIL 종합 (한 건에 위반 4개)

```
지출결의서 검토해주세요.

성명: 김수현 / 사원
지출일: 2026-06-10
목적: 야근 식대
지급: 개인카드
품의: 없음
야근 식대 5명 × 20,000원 = 100,000원 (편의점 간이영수증)
```

**예상 판정: ❌ FAIL (위반 4건)**

| 심각도 | 규칙 | 내용 |
|--------|------|------|
| 📋 누락 | R-05 | 품의 승인일 없음 → 사전 승인 여부 확인 불가 |
| 🔴 심각 | R-07 | 야근 식대 5인 한도 ₩75,000 초과 (청구 ₩100,000, 초과 ₩25,000 / 1인 한도 ₩15,000) |
| 🔴 심각 | R-01 | ₩30,000 초과(₩100,000)인데 **간이영수증** → 적격증빙 미비, 2% 가산세 |
| 🟡 주의 | R-06 | 개인카드 사용 사유 미기재 |

추가로 야근 식대는 **야근 일지 첨부 필수**(R-07) → 누락 시 보완 안내.

---

#### ✅ A-2. PASS (정상 결의서)

```
지출결의서 검토해주세요.

성명: 이서연 / 대리
소속: 개발팀
품의 승인일: 2026-06-09
지출일: 2026-06-10
1. 점심 식대 3인 ₩36,000 / 법인카드 / 신용카드 매출전표 첨부
2. 야근 식대 본인 ₩14,000 / 법인카드 / 야근일지 첨부
3. 야근 택시비 ₩19,000 / 법인카드 / 22:10~22:45, 강남역→방배동 자택 / 영수증 첨부
```

**예상 판정: ✅ PASS** — 모든 한도·증빙·기재 요건 충족.

---

#### 🔴 A-3. FAIL (야근 택시비 정보 누락)

```
지출결의서 검토해주세요.

성명: 박지훈 / 사원
품의 승인일: 2026-06-10
지출일: 2026-06-10
야근 택시비 ₩28,000 / 법인카드 / 영수증 첨부 (출발지·도착지·시간 미기재)
```

**예상 판정: ❌ FAIL** — R-08 야근 택시비는 **이용 시간·출발지·도착지 모두 명기** 필수. 누락.

---

#### 🔴 A-4. FAIL (접대비 + 공직자 한도)

```
지출결의서 검토해주세요.

성명: 정민호 / 과장
품의 승인일: 2026-06-09
지출일: 2026-06-10
거래처 접대 저녁 ₩240,000 / 법인카드
상대방: 구청 담당 공무원 박○○ 사무관 외 1명
```

**예상 판정: ❌ FAIL / ⚠️ 확인** — 상대방이 **공직자** → R-03 청탁금지법 음식물 한도(1인 ₩30,000) 적용.
2인 기준 ₩60,000 한도 대비 ₩240,000은 큰 폭 초과. R-10 상대방 회사·직위·성명 기재 필요.

---

### B. 정책 질문 (규격화 → 결정적 답변, 항상 동일)

```
야근 식대 1인 한도가 얼마예요?
```
→ **1인 ₩15,000 이하**, 야근 일지 첨부 필수, 저녁 식대와 중복 청구 불가 (R-07)

```
점심이랑 저녁 식대 한도는?
```
→ 점심 **₩13,000** / 저녁 **₩18,000** (1인 1일, R-07)

```
공직자 선물 한도는 얼마인가요?
```
→ 일반 **₩50,000**, 농수산물·가공품 **₩150,000**(명절 ₩300,000) (R-03)

```
견적은 몇 개 회사에서 받아야 하나요?
```
→ ₩100만 미만 단일 / ₩100만~2,000만 2개사 이상 / ₩2,000만 이상 3개사 또는 대표 승인 (R-11)

---

### C. 위반 수정 / 예외 처리 질문 (대화형)

```
법인카드가 없을 때 개인카드로 결제하면 어떻게 되나요?
```
→ R-06: 원칙은 법인카드. 카드 미수납 가맹점 등 **불가피한 사유**에 한해 예외, 결의서에 사유를 **구체적으로 기재**해야 인정.

```
₩30,000 넘는 금액인데 간이영수증만 있으면 어떻게 하죠?
```
→ R-01: ₩30,000 초과는 **적격증빙(세금계산서·신용카드 매출전표·현금영수증)** 필수. 간이영수증만이면 증빙불비가산세(2%) 대상 → 적격증빙 재발급/보완 필요.

```
사전 품의를 깜빡했는데 지금이라도 사후 처리할 수 있나요?
```
→ R-05: 사후 정산은 원칙적으로 거부. 불가피한 사정이면 **경영지원팀 사전 협의** 필요.

```
출장 신청서를 출장 다녀온 후에 제출했는데 어떻게 수정해야 하나요?
```
→ R-09: 출장 신청서는 **사전 제출** 원칙. 사후 제출은 사유서 + 경영지원팀 협의 필요, 출장보고서·증빙 영수증 정산.

---

### D. 공직자 접대

```
구청 공무원 2명이랑 밥을 먹었고 총 7만원 나왔어요. 문제 있나요?
```
→ 상대가 공직자이므로 **청탁금지법(R-03)** 적용. 음식물 한도는 **1인 ₩30,000**.
참석 인원 기준으로 1인당 금액을 확인해야 하며, 한도 초과 시 위반.
상대방 **소속·직위·성명 기재**(R-10)와 공직자 여부 확인이 필수.

---

### E. 멀티턴 (꼬리질문 — 직전 맥락 유지)

```
1턴: (A-1 결의서 붙여넣기)
2턴: 그럼 야근 식대는 얼마 줄여야 통과돼?
3턴: 개인카드는 왜 안돼?
4턴: 품의를 다시 올리면 통과될까?
```
→ 2~4턴은 **전체 보고서 양식을 반복하지 않고** 직전 결의서를 참조해 간결히 답변.

---

### F. 영수증 이미지 / 엑셀 입력

- **영수증 이미지(카카오 T 택시 등)**: `+`로 첨부 → 비전 추출(금액·시간·출발/도착·결제수단) + R-08 회사 룰 검증 → PASS/FAIL.
  텍스트를 함께 써서 ("이거 확인해줘") 영수증에 대해 이어서 질문 가능.
- **엑셀(.xlsx)**: 결의서 시트를 첨부 → 셀을 텍스트로 변환해 정책 검토.

## Part 3. 3분 시연 대본 (추천 순서)

1. **A-1 붙여넣기** → ❌ FAIL 4건 (배너 + 컬러 카드로 한눈에) — "한 건에 위반 4개를 규칙별로 잡아냅니다"
2. **꼬리질문** "야근 식대 얼마 줄여야 돼?" → 직전 결의서 참조한 간결 답변 — "멀티턴이 됩니다"
3. **정책 질문** "공직자 선물 한도?" → 규격화 질문은 **LLM 없이 결정적**으로 즉답
4. **영수증 이미지 첨부** (카카오 T 택시) → 비전 추출 + R-08 검증 PASS
5. **범위 밖** "오늘 날씨 어때?" → 정중히 거절 — "엉뚱한 답 안 합니다"
6. **A-2 붙여넣기** → ✅ PASS — "정상 건은 통과시킵니다"

> 포인트: **판정 숫자는 항상 동일**(결정적 규칙엔진), 영수증·대화는 LLM, 둘을 분리해 신뢰성과 자연스러움을 동시에 확보.
